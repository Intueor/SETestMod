РАБОТА С ЗАГОТОВКОЙ ПРОЕКТА МОДА

Для настройки рабочего и серверного компьютера:
1) На серверном компьютере устанавливаем Торч
2) Оба компьютера должны быть видимыми в домашней сети
3) На серверном компьютере сделать папку [где-то]torch-server\Instance\Mods общедоступной

В файле SendToStorage.bat:
1) Переменная ModsFolder содержит адрес общей папки модов на серверном компьютере
2) Переменная ModName содержит имя мода
3) Имена файлов *.cs и *.vdf должны соответствовать имеющимся в проекте

В файле [имя мода].vdf:
1) Поле "publishedfileid" сопоставляется значению SteamID мода, при первой загрузке в Стим - значение 0.
2) Поля "contentfolder" и "previewfile" сопоставляются адресу папки мода и превью соответственно, удобен сетевой тип адреса
3) Поле "visibility" позволяет установить видимость: 0 - открытый, 1 - только для друзей, 2 - скрытый, 3 - по ссылке
4) Имена папок в путях к моду должны соответствовать имени мода

При сборке проекта для VS проводится копирование необходимых файлов мода на компьютер сервера.

Для публикации мода после прихода файлов с рабочего компьютера после компиляции проекта:
1) Запускаем steamcmd.exe на серверном компьютере
2) Вводим login [имя пользователя] [пароль] и проходим двухфакторную аутентификацию
3) Вводим workshop_build_item \\[сетевое имя серверного компьютера]\Mods\[имя мода].vdf
4) Предыдущую команду можно повторять не выходя из аккаунта при необходимости обновления мода
